### Методы фреймворка

```
@single function

function core:kframe/method/entity_count {type:<type>}
    ⤷ Выводит количество сущностей указанного типа в мире

@single function

function core:kframe/method/on_start_reset
    ⤷ Сбрасывает поведение функции on_start для повторной активации при следующем входе в мир
```

### Активация неймспейсов фреймворка


**Неймспейсы** — это отдельные функциональные модули фреймворка, по сути как классы или компоненты.

По умолчанию они отключены для повышения производительности.
Чтобы активировать конкретный неймспейс, необходимо вручную подключить его load и tick функции:

```
@single on_load function

function inputs:load
```

```
@tick on_tick function

function inputs:tick
```

Без этого модуль **inputs** не будет работать корректно.

### 💡 Советы:

Подключайте только те неймспейсы, которые действительно нужны в целях улучшения производительности.

### Создание диалога

Эта команда создаёт новый объект диалога.
**Ограничений по количеству диалогов нет.**

```
@single function

function dialogue:create {node:,text:"",author:{text:""},step:1,selector:"@a",display:{color:gray},command:"",delay:1,stay:-1}
```

```
function dialogue:create {
    node:d1,
    text:" Привет Мир! Привет Мир!",
    author:{text:"[Koishem]",color:white},
    step:3,
    selector:"@a",
    display:{color:blue,italic:true},
    command:"playsound minecraft:entity.experience_orb.pickup master @a ~ ~ ~ 999",
    delay:3,
    stay: -1
}
```

```
node        | ID диалога
text        | Текст, который будет выведен
author      | SNBT-аргумент, вставляемый в начало диалога. (Можно вставить иконку персонажа через {text:"\uE000 "})
display     | SNBT-аргумент для настройки оформления текста (цвет, жирность, курсив и т.д.)
selector    | Селектор сущностей, которым будет показан диалог
    ⤷ При указании type=text_display..., вывод будет происходить через над сущностью
step        | Кол-во символов одновременно выводимых в диалоге
    ⤷ Если значение слишком большое — весь диалог появится сразу
delay       | 	Задержка между выводом символов (в тиках)
command     | Команда, выполняемая при каждом выводе символа
    ⤷ Если не нужно выполнять команду — используйте return fail
stay        | Задержка диалога после полного вывода. По окончанию stay таймера дисплеи очищаются
    ⤷ Если не нужна задержка - используйте -1
```

### Создание пресета диалога
Пресеты - готовые настройки для создания диалогов. Значительно укорачивают код

```
function dialogue:create_preset {name:preset1,author:{color:red,text:""},step:1,delay:1,selector:"@a",stay:60,command:"",display:{color:white}}
```

```
function dialogue:create_preset {

    author:{color:red,text:""},
    step:1,
    delay:1,
    selector:"@a",
    stay:60,
    command:"",
    display:{color:white}

```

```
name        | ID пресета
author      | SNBT-аргумент, вставляемый в начало диалога. (Можно вставить иконку персонажа через {text:"\uE000 "})
display     | SNBT-аргумент для настройки оформления текста (цвет, жирность, курсив и т.д.)
selector    | Селектор сущностей, которым будет показан диалог
    ⤷ При указании type=text_display..., вывод будет происходить через над сущностью
step        | Кол-во символов одновременно выводимых в диалоге
    ⤷ Если значение слишком большое — весь диалог появится сразу
delay       | 	Задержка между выводом символов (в тиках)
command     | Команда, выполняемая при каждом выводе символа
    ⤷ Если не нужно выполнять команду — используйте return fail
stay        | Задержка диалога после полного вывода. По окончанию stay таймера дисплеи очищаются
    ⤷ Если не нужна задержка - используйте -1
```


### Удаление пресета

```
function dialogue:delete_preset {name:<preset>}
```

### Удаление диалога

Удаляет существующий диалог по ID.

```
@single function

function dialogue:delete {node:<node>}
```

### Сброс  диалога

Сбрасывает прогресс диалога, позволяя начать его заново.

```
@single function

function dialogue:reset {node:<node>}
```

### Вывод диалога

Активирует вывод диалога по шагам.
Рекомендуется вызывать в тике, пока диалог не завершится.

```
@tick function

function dialogue:iterate {node:<node>}
```

### 💡 Советы:

Можно выводить диалоги сразу нескольким игрокам, используя селекторы.

Команды, выполняемые при выводе каждого символа **(command)**, подходят для звуковых эффектов или других реакций — ставьте "return fail", если команды не нужны.

Вызывайте **iterate** в тике для поэтапного вывода, пока диалог не будет полностью завершён.

# Сохранения

Система сохранений — это динамичная структура с хранилищами на 10 ячеек, предназначенная для временного хранения значений scoreboard. Позволяет сохранять, изменять и восстанавливать значения, привязанные к игрокам или фейковым сущностям.

- [Активация сохранений](#активация-сохранений)

- [Удаление ячеек](#удаление-ячеек)

### Создание сохранения

Создаёт новое хранилище на 10 ячеек.

```
@single function

function quicksave:create {node:<node>}
```

### Заполнение ячеек

Метод **append** записывает данные в первую свободную ячейку.
Если ячейки были удалены, **append** не использует освободившиеся слоты.

```
@single function

function quicksave:append {node:<node>,score:"<score>",obj:"<objective>"}
```

```
function quicksave:append {
    node:save1,
    score:".FakePlayer",
    obj:"testing"
}
```

```
node        | ID сохранения
score       | Селектор или имя фэйкового игрока
obj         | Название scoreboard objective
```

### Принт сохранений

Показывает в чат ID хранилища и значения всех ячеек.

```
@single function

function quicksave:print {node:<node>}
```

### Активация сохранений

Восстанавливает значения scoreboard из хранилища обратно на указанных игроков/фэйк игроков

```
@single function

function quicksave:activate {node:<node>}
```

### Редактирование ячеек

Изменяет содержимое конкретной ячейки по её ID (0–9).

```
function quicksave:set {node:<node>,id:<id>,score:"<score>",obj:"<objective>"}
```

```
function quicksave:set {
    node:save1,
    id:1,
    score:".val4",
    obj:Test
}
```

```
node        | ID сохранения
id          | Номер ячейки (от 0 до 9)
score       | Селектор или имя фэйкового игрока
obj         | Название scoreboard objective
```

### Удаление ячеек

Очищает ячейку по её номеру и ID хранилища.

```
function quicksave:pop {node:<node>,id:<id>}
```

### 💡 Советы:

Используйте сохранения для быстрого хранения и восстановления значений в скорбордах, например, для чекпоинтов или временных состояний.

Всегда проверяйте, что вы создаёте уникальный ID сохранения (node), чтобы не перезаписать важные данные.

Метод **append** заполняет только свободные ячейки — если удаляли ячейки, учтите, что они не будут заполнены повторно. Для перезаписи используйте метод **set**

Для дебага выводите содержимое сохранений через **print**, чтобы видеть текущие данные

# Инпуты

Система инпутов отслеживает нажатия клавиш игроками:
W, A, S, D, SPACE, CTRL, SHIFT, слоты хотбара 0-9

Инпуты можно замыкать (lock) и отмыкать (unlock), включая/отключая их отслеживание для конкретных игроков.

- [Методы](#методы)
- [Замыкание](#замыкание)
- [Отмыкание](#отмыкание)
- [Триггеры](#триггеры)
- [Одиночные триггеры](#одиночные-триггеры)

### Методы

Утилиты для работы с инпутами у определённого игрока:

```
@single function
execute as <@player> run function inputs:method/toggle_lock {arg:<input>}
    ⤷ Переключает статус отслеживания конкретного инпута с 0 на 1 и наоборот

@single function
execute as <@player> run function inputs:method/check_chat
    ⤷ Показывает замкнутые инпуты в чате

@single function
execute as <@player> run function inputs:method/check_actionbar
    ⤷ Показывает замкнутые инпуты в actionbar

@tick function
execute as <@player> run function inputs:method/check_trigger
    ⤷ Отображает нажатия клавиш в режиме actionbar (в реальном времени)
```

### Замыкание

Отключает отслеживание выбранного инпута для игрока

```
@single function

execute as <@player> run function inputs:lock {arg:<input>}
```

### Отмыкание

Повторно включает отслеживание замкнутого инпута

```
@single function

execute as <@player> run function inputs:unlock {arg:<input>}
```

### Триггеры

Используются для условного выполнения команд, если игрок удерживает или нажимает нужную клавишу

```
@tick function

execute as <@player> if function inputs:trigger/<input>
```

### Одиночные триггеры

Отличаются от обычных: команда выполняется только один раз при нажатии, даже если клавиша удерживается.

**Важно**: в одной функции можно использовать только один одиночный триггер на один тип инпута, иначе сработает лишь первый

```
@tick function

execute as <@player> if function inputs:trigger/single/<input>
```

### 💡 Советы:

Используйте замыкание **(lock)** для временного отключения обработки конкретных кнопок у игроков — удобно для кастомных меню или механик.

Не забывайте отмыкать **(unlock)** инпуты, чтобы вернуть нормальную работу клавиш.

Внимательно следите за вызовом одиночных триггеров — при их использовании в одной функции должен быть только один триггер на определённый инпут.

Используйте методы проверки **(check_chat, check_actionbar)** для отладки и визуализации, какие инпуты сейчас замкнуты

Помните, что для работы инпутов нужно подключать неймспейс **inputs** в **load** и **tick**

# Таймеры

**K-Framework** содержит гибкую систему таймеров, построенную на формате ЧЧ:ММ:СС:ТТ (часы : минуты : секунды : тики)

- [Удаление таймера](#удаление-таймера)
- [Настройка таймера](#настройка-таймера)
- [Активация таймера](#активация-таймера)
- [Обнуление таймера](#обнуление-таймера)
- [Вывод таймера](#вывод-таймера)

### Создание таймера

```
@single function

function timers:create {node:,t:0,h:0,m:0,s:0,end_h:0,end_m:0,end_s:0,end_t:0,single:0,command:""}
```

```
function timers:create {
    node:t1,
    t:0,
    h:0,
    m:0,
    s:57,
    end_h:0,
    end_m:1,
    end_s:0,
    end_t:0,
    single:1,
    command:"say hi"
}
```

```
node                        | ID таймера
t                           | Начальные тики
h, m, s                     | Начальные часы, минуты, секунды
end_h, end_m, end_s, end_t  | Время окончания таймера 
single                      | 1 — команда запускается один раз, 0 — циклично
command                     | Команда которая запускается при окончании таймера
    ⤷ Если не нужно выполнять команду — используйте return fail
```

### Удаление таймера

Удаляет таймер с указанным ID

```
@single function

function timers:delete {node:<node>}
```

### Настройка таймера

Позволяет изменить любое поле существующего таймера

```
@single function

function timers:set {node:,arg:,value:}
```

```
function timers:set {
    node:t1,
    arg:s,
    value:5
}
```

```
node        | ID таймера
arg         | Аргумент для изменения
    ⤷ Можно менять t, s, m, h, end_t, end_s, end_m, end_h, command
value       | Значение для аргумента
    ⤷ При смене command ставьте лапки {id:,arg:command,value:"say hello world"}
```

### Активация таймера

Активирует таймер на каждом тике:

```
@tick function

function timers:iterate {node:<node>}
```

### Обнуление таймера

Сбрасывает таймер к начальному состоянию

```
@single function

function timers:reset {node:<node>}
```

### Вывод таймера

Показывает текущее значение таймера в actionbar

```
@tick function

function timers:check with storage timers <node>
```

### 💡 Советы:

При создании таймера тщательно задавайте начальное и конечное время, чтобы избежать неожиданных циклов или слишком долгих ожиданий.

Для динамической смены параметров таймера используйте метод set, чтобы не пересоздавать таймер полностью.

Не забывайте сбрасывать таймеры через reset, если хотите начать отсчёт заново без пересоздания объекта.

### Создание триггера

Создание триггера в конкретной позиции

```
@single function

function 3d_triggers:create {node:,dx:,dy:,dz:,x:,y:,z:,command:"",single:,selector:"",color:""}
    ⤷ Команда вводится в функции датапака

function 3d_triggers:create_at_player {node:,dx:,dy:,dz:,single:,command:"",selector:"",color:""}
    ⤷ Команда вводится в чат игры
```

Создание триггера на месте игрока

```
function 3d_triggers:create_at_player {
    node:t1,
    dx:0,
    dy:0,
    dz:0,
    single:0,
    command:"say hello world",
    selector:"@a",
    color:"red"
}
```

```
function 3d_triggers:create {
    node:t1,
    dx:1,
    dy:1,
    dz:1,
    x:8,
    y:-60,
    z:10,
    command:"say hi",
    single:1,
    selector:"@a",
    color:"lime"
}
```

```
node                | ID триггера
dx, dy, dz          | Размеры триггера (в блоках), от якоря до положительной стороны (0 равен 1)
x, y, z             | Позиция якоря (нижний угол куба)
command:""          | Команда, запускаемая при пересечении
single:             | 1 — срабатывает один раз, 0 — срабатывает циклично
selector:""         | Селектор сущностей, на которых реагирует триггер
color:""            | Цвет триггера при дебаге (debug_on)
```

### Проверка триггера

Активирует проверку соприкосновения сущностей с указанным триггером. Обязательно запускать для активации, дебага и ручного перемещения

```
@tick function

function 3d_triggers:check {node:<node>}
```

### Дебаг триггера

Включает/отключает визуализацию триггера. Работает только если запущена проверка **check**.

```
@single/tick function

function 3d_triggers:debug_on

function 3d_triggers:debug_off
```

### Перемещение триггера вручную

Позволяет привязать позицию триггера к игроку. Работает только при включённой проверке **check**.

```
@single function

 function 3d_triggers:move
    ⤷ Вызывается в чате — триггер начинает следовать за игроком
```

### Обнуление триггера

Сбрасывает single-триггер, чтобы он снова мог сработать

```
@single function

function 3d_triggers:reset {node:<node>}
```

### Удаление триггера

Полностью удаляет указанный триггер

```
@single function

function 3d_triggers:delete {node:<node>}
```

### 💡 Советы:

Для отладки используйте режим дебага, чтобы визуально видеть границы триггеров

Не забывайте обнулять триггеры с помощью reset для повторной активации
